# Disease Ontology (DO) Operational Reference

---

## Setup

No API key or account required. The OBO file is a public GitHub raw download.

**databases.yaml entry:**
```yaml
disease_ontology:
  enabled: true
  args: {}
```

No `args` are needed; the download URL is a fixed constant in the parser.

**Requires:** `obonet` Python package (`pip install obonet`).

**Prerequisites:** `slim-terms.tsv` must exist at `data/raw/diseaseontology/slim-terms.tsv` before parsing. Generate it by running:
```bash
python src/generate_disease_slim.py
```
If this file is absent, the slim filter is skipped with a warning and all scope-matching DO terms are included instead.

## Source File

`doid.obo` — the full Human Disease Ontology in OBO format.

**Download URL:**
```
https://raw.githubusercontent.com/DiseaseOntology/HumanDiseaseOntology/main/src/ontology/doid.obo
```

Points to the `main` branch head; content changes with every DO release. No version pin in the URL.

### OBO Term Structure

Terms are `[Term]` stanzas. Fields used:

| OBO field | Description |
|---|---|
| `id:` | Disease Ontology ID (e.g. `DOID:10652`) |
| `name:` | Disease preferred name |
| `def:` | Definition in OBO format: `"text." [source1, source2]`; plain text extracted, trailing `[...]` block removed |
| `xref:` | External cross-references; `UMLS_CUI:` and `MESH:`/`MSH:` prefixes are extracted |
| `subset:` | Subset membership; `TopNodes_DOcancerslim` is used by `generate_disease_slim.py` |
| `is_obsolete:` | `true` — obsolete terms are excluded |

Only `DOID:`-prefixed terms are retained. Non-DOID entries (e.g. `OMIM:`) are skipped.

### xref Prefixes

| Prefix in OBO file | Field extracted | Notes |
|---|---|---|
| `UMLS_CUI:` | `umls_cui` column | Underscore in prefix, not colon |
| `MESH:` | stripped and discarded (used for scope match) | Bare D-code after prefix removal |
| `MSH:` | stripped and discarded (used for scope match) | Alternate MeSH prefix; treated identically |

### Symptom Extraction

The parser scans each term's raw `def:` string for `has_symptom` or `has symptom` clauses using a regex (`has[_ ]symptom\s+<term>`). Matched symptom names are returned as a pipe-separated string in the `symptoms` output column. This is a text-extraction heuristic, not a structured OBO field.

## slim-terms.tsv — Disease Slim Filter

`data/raw/diseaseontology/slim-terms.tsv` is a locally curated TSV that gates which DOIDs the parser considers. Generated by `src/generate_disease_slim.py` from two sources:

| Source | Selection criterion | `source` column value | `pathophysiology` value |
|---|---|---|---|
| `doid.obo` `TopNodes_DOcancerslim` subset | Tagged with `subset: TopNodes_DOcancerslim` | `TopNodes_DOcancerslim` | `neoplastic` |
| GWAS Catalog + MONDO | MONDO terms with >30 unique GWAS studies, mapped to DOID via MONDO xrefs | `GWAS_Catalog` | *(empty)* |

Cancer slim terms overwrite GWAS-derived terms for overlapping DOIDs. The script downloads `mondo.obo` from OBOLibrary and the GWAS Catalog studies file from EBI if not already cached.

**slim-terms.tsv columns:** `doid`, `name`, `source`, `pathophysiology`

---

## Two-Stage Filtering at Parse Time

A DO term passes only if **both** conditions are met:

1. **Slim filter:** DOID is listed in `slim-terms.tsv`. If the file is absent, this condition is skipped.
2. **Scope filter:** DOID, UMLS CUI, MeSH ID, or disease name substring matches any entry in `config/project.yaml` → `disease_scope` (`doid_ids`, `umls_cuis`, `mesh_ids`, `primary_terms`). If no scope is configured, all terms pass.

## Known Gotchas

**URL tracks `main` branch head, not a versioned release** — the DO team releases new versions regularly. The downloaded `doid.obo` reflects whatever is on `main` at download time. Record the download date or the commit SHA for reproducibility.

**`slim-terms.tsv` is a project artifact, not a source download** — it must be generated before parsing, using `src/generate_disease_slim.py`. This script itself downloads `mondo.obo` and the GWAS Catalog studies TSV. These are separate prerequisites; if either is unavailable, the slim file will be incomplete.

**`UMLS_CUI:` prefix uses an underscore** — the OBO xref format for UMLS is `UMLS_CUI:C1234567`, not `UMLS:C1234567`. Lookups against the `umls_cui` output column will fail if the caller expects the colon-only form.

**Symptom extraction is regex-based on free text** — `has_symptom` terms are extracted from the definition prose, not from a structured OBO relationship field. Non-standard phrasing or multi-word symptoms spanning punctuation may be missed or truncated.

**Empty disease scope in `project.yaml` admits all slim-matching terms** — if `disease_scope` is not configured, filter (b) is a no-op and every term in the slim passes. Verify `project.yaml` scope before running if a narrow disease set is expected.
